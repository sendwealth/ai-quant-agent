## 🔄 工作流程

### 1. 数据更新流程（每周日18:00）
```bash
cron → fetch_tushare_auto.py → 验证数据 → 保存CSV
```

**检查点**:
- 数据格式（整数/字符串）
- 数据完整性（>100天）
- 数据新鲜度（<7天）

### 2. 策略分析流程（每日8:50）
```bash
cron → run.py → 加载数据 → 生成信号 → 保存结果
```

**输出**:
- BUY/SELL/HOLD信号
- 价格预测
- 风险预警

### 3. 交易执行流程（手动/自动）
```bash
信号 → paper_trading.py → 风险检查 → 执行交易 → 保存状态
```

**风控**:
- 止损: -6%
- 止盈1: +10%卖50%
- 止盈2: +20%清仓
- 仓位限制: 单只≤30%, 总仓位≤80%

### 4. 监控流程（每30分钟）
```bash
心跳 → heartbeat_check.py → 检查状态 → 决定是否通知
```

**检查项**:
- 数据健康
- 账户状态
- 风险告警

## 🛡️ 风控体系

### 三层风控

**Level 1: 数据层**
- 数据健康检查（新鲜度、完整性）
- 异常检测（缺失值、异常值）
- 自动修复（重新获取）

**Level 2: 策略层**
- 参数优化（网格搜索）
- 多维度验证（Walk-Forward、蒙特卡洛）
- 压力测试（不同市场环境）

**Level 3: 执行层**
- 止损/止盈
- 仓位限制
- 熔断机制

### 风险等级

| 等级 | 触发条件 | 动作 |
|------|---------|------|
| 🟢 OK | 无异常 | 正常运行 |
| 🟡 WARNING | 轻微风险 | 记录日志 |
| 🔴 ERROR | 严重风险 | 立即通知 |
| ⛔ CRITICAL | 系统故障 | 停止交易 |

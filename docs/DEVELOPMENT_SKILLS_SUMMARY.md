#### 4.1 安全事件处理

**问题**: .env文件被提交到Git

**解决方案** (2分钟):
```bash
# 1. Git历史重写
git filter-branch --force --index-filter \
  'git rm --cached --ignore-unmatch .env' \
  --prune-empty --tag-name-filter cat -- --all

# 2. 创建pre-commit hook
echo "#!/bin/bash
if git diff --cached --name-only | grep -E '\.env'; then
  echo "❌ 禁止提交.env文件"
  exit 1
fi" > .git/hooks/pre-commit

# 3. 强制推送
git push origin main --force
```

**核心技能**:
- ✅ Git历史重写 (filter-branch)
- ✅ 安全机制设计
- ✅ 快速响应

## 🎯 开发方法论

### 1. 问题驱动开发 (PDD)

**流程**:
1. 识别问题
2. 分析根本原因
3. 设计解决方案
4. 快速实施
5. 验证结果
6. 文档记录

**应用示例**:
- **问题**: 回测使用旧股票
- **分析**: 硬编码列表 + 不运行agents
- **方案**: 从动态选股读取 + 自动运行
- **实施**: 创建complete_backtest.py
- **验证**: 测试通过 (100%)
- **文档**: COMPLETE_BACKTEST_FIX.md

---

### 2. 渐进式重构

**原则**:
- 小步快跑
- 持续测试
- 保持可用性
- 随时回滚

**应用**:
```
Phase 1: 修复单个Agent (1个/次)
Phase 2: 优化数据源 (1个/次)
Phase 3: 简化项目 (1个目录/次)
Phase 4: 清理文档 (1类/次)
Phase 5: 修复Git (1个问题/次)
```

**结果**:
- ✅ 每步都有进展
- ✅ 不会引入大bug
- ✅ 可以快速定位问题

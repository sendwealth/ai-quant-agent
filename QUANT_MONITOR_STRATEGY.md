## ⏰ 定时任务配置

### 1. 实时监控 (心跳触发)

**触发方式**: HEARTBEAT.md 中添加检查项
**频率**: 约30分钟（跟随心跳）
**检查内容**:
- 当前持仓状态
- 是否触及止损/止盈线
- 是否有新交易信号

**配置位置**: `~/clawd/HEARTBEAT.md`

```markdown
## 6️⃣ 量化策略监控
- 文件: `~/clawd/ai-quant-agent/data/auto_portfolio.json`
- 动作:
  - 检查持仓盈亏
  - 检查止损/止盈触发
  - 检查是否有新信号
  - 异常时通知Master
```

---

### 2. 每日监控 (Cron)

**时间**: 每个交易日 8:50 (开盘前)
**任务**: `daily_monitor.py`

**功能**:
1. 更新股票价格数据 (昨日收盘)
2. 运行策略生成信号
3. 市场情绪分析
4. 生成当日操作建议
5. 保存到文件供心跳读取

**Cron配置**:
```bash
# 每个交易日 8:50 运行 (盘前分析)
50 8 * * 1-5 cd ~/clawd/ai-quant-agent && python examples/daily_monitor.py >> logs/pre_market_$(date +\%Y\%m\%d).log 2>&1
```

---

### 3. 每周监控 (Cron)

**时间**: 每周日 10:00
**任务**: `weekly_analysis.py`

**功能**:
1. 本周交易回顾
2. 策略回测验证
3. 参数优化建议
4. 下周操作建议
5. 生成周报

**Cron配置**:
```bash
# 每周日 10:00 运行
0 10 * * 0 cd ~/clawd/ai-quant-agent && python examples/weekly_analysis.py >> logs/weekly_$(date +\%Y\%m\%d).log 2>&1
```

## 📱 通知策略

### 通知渠道
- **主渠道**: 飞书 (当前)
- **备用渠道**: 微信/邮件

### 通知优先级

| 级别 | 场景 | 方式 |
|------|------|------|
| 🔴 紧急 | 止损触发、异常波动 | 立即通知 |
| 🟡 警告 | 接近止损/止盈、信号变化 | 当日通知 |
| 🟢 信息 | 每日报告、每周分析 | 定时推送 |
| ⚪ 静默 | 正常状态、无信号 | 不通知 |

## ⚠️ 风险控制

### 异常处理
1. **数据源异常**: 使用缓存数据，记录错误
2. **API限流**: 自动降级频率，等待恢复
3. **网络故障**: 本地缓存，重试机制
4. **策略异常**: 停止交易，人工介入

### 安全措施
1. 所有操作记录日志
2. 关键操作需确认
3. 异常自动熔断
4. 定期备份配置
